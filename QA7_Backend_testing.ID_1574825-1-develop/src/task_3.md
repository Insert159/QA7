## Анализ Яндекс Музыки

### Гипотеза о типе БД

**Основным типом БД для Яндекс Музыки является гибридный подход: реляционная БД + колоночная NoSQL**

### Функция №1: Авторизация и профиль

| Параметр | Описание |
|----------|----------|
| Данные | ID, email, пароль, имя, подписка |
| Тип БД | Реляционная (PostgreSQL) |
| Почему | Чёткая структура, транзакции, целостность |

### Функция №2: Прослушивание и история

| Параметр | Описание |
|----------|----------|
| Данные | user_id, track_id, timestamp, duration |
| Тип БД | Колоночная (Cassandra / ClickHouse) |
| Почему | Огромный объём данных, только запись, агрегация |

### Функция №3: Плейлисты и избранное

| Параметр | Описание |
|----------|----------|
| Данные | Плейлист → треки (связь многие-ко-многим) |
| Тип БД | Реляционная (PostgreSQL) |
| Почему | JOIN-запросы, ссылочная целостность |

### Функция №4: Рекомендации

| Параметр | Описание |
|----------|----------|
| Данные | "Кто слушал X, слушает и Y" |
| Тип БД | Графовая + колоночная |
| Почему | Поиск связей + агрегация статистики |
