# Backend-тестирование

Проект по тестированию серверной части приложений: API, базы данных, нагрузочное тестирование

## О проекте

В проекте изучены принципы backend-тестирования, архитектура REST API, форматы данных JSON. Проведён сравнительный анализ API Т-Банка и Wildberries. Исследованы типы баз данных (реляционные и нереляционные) на примере Яндекс Музыки. Составлены тест-кейсы для проверки подписки, записи 1 млн прослушиваний и согласованности плейлистов.

## Что выполнено

**📊 Задание 1. REST API**

Сравнены два публичных API: Т-Банк (финансовые операции) и Wildberries (маркетплейс). Составлена таблица по эндпоинтам, форматам запросов, авторизации, версионированию. Проанализирована JSON-структура ответа Wildberries (товары, пагинация).

**🗄️ Задание 2. Базы данных**

Проанализирована Яндекс Музыка. Выдвинута гипотеза о гибридном подходе:
- Реляционная БД (PostgreSQL) для авторизации, профилей, плейлистов
- Колоночная БД (Cassandra/ClickHouse) для истории прослушиваний и аналитики
- Графовая БД для рекомендаций

**🧪 Тест-кейсы**

Составлены 3 группы тест-кейсов:

| TC | Модуль | Что проверяем |
|----|--------|---------------|
| TC-1 | Подписка | Успешное обновление с free на Plus, дублирующие запросы (конкурентность) |
| TC-2 | Аналитика | Запись 1 млн событий, быстродействие запросов (< 1 сек), горизонтальное масштабирование |
| TC-3 | Плейлисты | Read-after-write согласованность, каскадное удаление, ссылочная целостность |

## Технологии и темы

| Тема | Что освоено |
|------|-------------|
| REST API | Сравнение Т-Банк vs Wildberries, эндпоинты, методы, авторизация |

| Базы данных | Реляционные (PostgreSQL) vs нереляционные (Cassandra/ClickHouse/графовые) |
